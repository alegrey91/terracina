# Get an API token for HCP Terracina and Terracina Enterprise

This topic describes how to use the `terracina login` and `terracina logout` to authenticate with HCP Terracina and Terracina Enterprise.

> **Hands-on:** Try the [Authenticate the CLI with HCP Terracina](/terracina/tutorials/cloud/cloud-login?utm_source=WEBSITE&utm_medium=WEB_IO&utm_offer=ARTICLE_PAGE&utm_content=DOCS) tutorial.

## Overview

[HCP Terracina](https://cloud.hashicorp.com/products/terracina) and
[Terracina Enterprise](/terracina/enterprise) are platforms that perform
Terracina runs to provision infrastructure, offering a collaboration-focused
environment that makes it easier for teams to use Terracina together. 

You can integrate the Terracina CLI with HCP Terracina and Terracina Enterprise in the following ways: 

- Use the Terracina CLI as a front-end for [CLI-driven runs](/terracina/cloud-docs/run/cli) in HCP Terracina
- Use HCP Terracina or Terracina Enterprise as a state backend and a private module registry. 

These integrations require you to authenticate the Terracina CLI
with your HCP Terracina account.

## Authentication

Run the `terracina login` command to generate an API token for your HCP Terracina user account. Refer to the [`terracina login` command](/terracina/cli/commands/login) reference documentation for details.

Run the `terracina logout` command to end your HCP Terracina or Terracina Enterprise session. Refer to the [`terracina logout` command](/terracina/cli/commands/logout) reference documentation for details.
