# `terracina force-unlock` command

This topic provides reference information about the `terracina force-unlock` command. This command manually unlocks the state for the defined configuration.

This command removes the lock on the state for the current configuration. The behavior of this lock is dependent
on the backend being used. Local state files cannot be unlocked by another
process. The `terracina force-unlock` command does not modify your infrastructure. 

## Usage

Usage: `terracina force-unlock [options] LOCK_ID`

Manually unlock the state for the defined configuration.

This will not modify your infrastructure. This command removes the lock on the
state for the current configuration. The behavior of this lock is dependent
on the backend being used. Local state files cannot be unlocked by another
process.

Options:

- `-force` -  Don't ask for input for unlock confirmation.
